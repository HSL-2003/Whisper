# Custom Agent Rules for Whisper Workspace

## Code Modification Protocol: Plan Before You Code
Before modifying any files or running commands that alter the codebase, you MUST write a clear, step-by-step implementation plan for the user to review. Do not make any edits to the project code files until the user explicitly reviews the plan and gives confirmation to proceed.

## Detailed Code Explanation Protocol
Whenever you propose code modifications, you MUST provide a detailed explanation of:
1. What the code does in detail.
2. Where the modified functions are called in the codebase.
3. The exact purpose of each function.
4. The rationale for introducing or modifying these functions.
5. A detailed analysis of each function's inputs, outputs, and internal logic.

## Output Verification Protocol
Before proposing any code modifications to the user, you MUST verify, test, or dry-run the logic and implementation details at least 2 times to guarantee correctness, compatibility, and ensure no regressions are introduced before submission.
